# Story 6.1: Content Library

## Goal

The Library tab shows a grid of all content (newsletters, LinkedIn posts, blogs) with filtering, status badges, and the ability to open content in the preview.

## Dependencies

- Story 2.1 (file watcher / content listing)
- Story 2.2 (preview renderer)

## Files to Create/Edit

```
src/renderer/components/content-library.tsx  # Grid view container
src/renderer/components/content-card.tsx     # Card: thumbnail, title, type, status, date
src/renderer/components/filter-bar.tsx       # Type + status filters
src/renderer/hooks/use-content.ts           # Update: library data
```

## Card Contents

- **Thumbnail**: first 200px of HTML content (or a placeholder icon)
- **Title**: from newsletter.json subject, post.json title, or blog frontmatter
- **Type badge**: Newsletter / LinkedIn / Blog
- **Status badge**: Draft / Published
- **Date**: creation date from metadata
- **Click**: opens in preview pane

## Filters

- By type: All / Newsletter / LinkedIn / Blog
- By status: All / Draft / Published
- Search by keyword (searches title and file paths)

## Acceptance Criteria

- [ ] Library tab shows grid of content cards
- [ ] Cards show title, type badge, status badge, date
- [ ] Filters work (type + status)
- [ ] Clicking a card opens it in preview
- [ ] Empty state for no content
- [ ] Refreshes when content is created/deleted
