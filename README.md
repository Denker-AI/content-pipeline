# Content Pipeline

Desktop app wrapping Claude Code with visual content creation tools. Create, preview, and publish LinkedIn posts, newsletters, and blog articles — all from one window.

## Features

- **Terminal + Preview**: Split-pane layout with a real Claude Code terminal on the left and live content preview on the right
- **Pipeline Sidebar**: Organize content pieces by type (LinkedIn, newsletter, blog) with stage tracking (idea → draft → review → published)
- **Component Browser**: Browse and preview React components generated by Claude
- **Capture Tools**: Screenshot and video capture with size presets (LinkedIn carousel, OG image, newsletter hero)
- **Annotations**: Pin comments on the preview and send revision instructions back to Claude
- **One-Click Publishing**: Publish to LinkedIn (OAuth), send newsletters via Resend, or POST blog content to any webhook
- **Auto-Updates**: Built-in update checker for new releases

## Installation

### macOS (Homebrew)

```bash
brew install --cask content-pipeline
```

### macOS / Linux / Windows (Direct Download)

Download the latest release for your platform:

- **macOS**: `.dmg` (Apple Silicon + Intel)
- **Linux**: `.AppImage`
- **Windows**: `.exe` installer

[Download from GitHub Releases](https://github.com/denkerapp/content-pipeline/releases)

### npm (Global CLI Launcher)

```bash
npm install -g content-pipeline
content-pipeline
```

The npm package provides a `content-pipeline` command that launches the installed desktop app. You still need the app installed via one of the methods above.

## Prerequisites

- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) installed and authenticated
- Node.js 18+ (for development)

## Development

```bash
# Install dependencies
bun install

# Start in development mode (Electron + Vite HMR)
bun dev

# Run checks
bun lint
bun typecheck
bun test:run

# Build for production
bun run build

# Package as installer (.dmg / .AppImage / .exe)
bun package
```

## Architecture

```
src/
├── main/           # Electron main process (Node.js)
│   ├── index.ts    # App entry, window management
│   ├── pty.ts      # Terminal PTY (node-pty)
│   ├── ipc.ts      # IPC handler registry
│   └── ...
├── renderer/       # Electron renderer (React + Vite)
│   ├── App.tsx     # Root: split pane layout
│   ├── components/ # UI components
│   ├── hooks/      # React hooks
│   └── styles/     # Tailwind CSS
├── shared/
│   └── types.ts    # Shared types (IPC messages, content metadata)
npm/                # npm global CLI wrapper
Caskfile            # Homebrew cask formula
```

## Publishing Integrations

| Channel | Method | Config |
|---------|--------|--------|
| LinkedIn | OAuth2 API | Access token in Settings |
| Newsletter | Resend API | API key in Settings |
| Blog | Webhook POST | URL in project settings |
